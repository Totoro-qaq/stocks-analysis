# 美股 50 支股票投资策略分析 MVP 项目计划

## 1. 项目目标

本项目要做一个可复现的美股投资策略分析 MVP，用于课程作业和展示。

系统目标：

- 使用真实市场数据，而不是为了作业随便构造数据。
- 将原始数据同时保存为 `data/raw/` 下的 CSV 和 MySQL 数据库。
- 围绕 50 支美股构建投资组合分析。
- 引入纳斯达克100、标普500科技板块、标普500、VIX 恐慌指数、短期无风险利率等基准和风险代理变量。
- 计算收益、风险、回撤、β、波动率、相关性、夏普比率等指标。
- 实现马科维茨均值-方差模型。
- 使用时间序列前向验证，而不是普通随机交叉验证。
- 加入过拟合校验和统计显著性分析。
- 做可视化 Dashboard，便于展示。
- 使用 DeepSeek 辅助生成投研报告，但不让大模型直接替代量化计算。

- 从 State Street 官方 XLK 当前持仓中筛选。
- 筛选依据包括：
  - 价格数据覆盖率
  - 当前是否仍有可用行情
  - XLK 持仓权重
  - 近一年美元成交额流动性

已纳入的基准和风险代理：

- `^GSPC`：标普500指数
- `^NDX`：纳斯达克100指数
- `QQQ`：纳斯达克100可交易 ETF 代理
- `XLK`：标普500科技板块可交易 ETF 代理
- `^VIX`：CBOE 恐慌指数
- `^IRX`：13 周美国国债收益率，作为短期无风险利率代理

重要限制：

- 当前股票池是真实、可追溯来源的数据。
- 但它是“当前 XLK 成分股向历史回看”，不是历史每个时间点的真实成分股。
- 因此它可以避免随意选股，但不能完全排除生存者偏差。
- 如果要完全控制生存者偏差，需要 CRSP、Norgate、Polygon、Sharadar 等包含退市股票和历史成分的数据源。

## 4. 为什么不用普通交叉验证

金融时间序列不能随便随机打乱做交叉验证。

原因：

- 会把未来信息泄漏到训练集。
- 会高估策略效果。
- 会导致过拟合结果看起来很漂亮。

本项目采用：

- 滚动窗口验证
- 扩展窗口验证
- Walk-forward 前向验证

示例：

```text
训练集：2020-01-02 到 2022-12-31
测试集：2023-01-01 到 2023-06-30

训练集：2020-07-01 到 2023-06-30
测试集：2023-07-01 到 2023-12-31
```

每一轮中，组合权重只用训练期数据估计，然后应用到未来测试期。

## 5. 策略模型

### 等权组合

目的：

- 作为最简单的基准。
- 很容易解释。
- 不容易过拟合。

公式：

```text
每只股票权重 = 1 / 股票数量
```

### 马科维茨最小方差组合

目的：

- 降低组合波动率。
- 展示经典投资组合理论。

目标函数：

```text
最小化 w.T * Sigma * w
约束：
sum(w) = 1
0 <= w_i <= 单股最大权重
```

### 马科维茨最大夏普组合

目的：

- 最大化风险调整后收益。

目标函数：

```text
最大化 (组合收益率 - 无风险利率) / 组合波动率
```

实现方式：

- 使用 `scipy.optimize.minimize`
- 使用复权收盘价计算日收益率
- 年化收益率和协方差矩阵
- 不允许做空
- 加单股最大权重约束

## 7. 过拟合检查

本项目会加入以下检查：

1. Walk-forward 样本外测试。
2. 优化组合与等权组合对比。
3. 限制单股权重，避免优化器集中押注少数股票。
4. 必要时使用协方差收缩。
5. 重点展示测试期表现，而不是只展示全样本结果。
6. 使用 Bootstrap 置信区间。
7. 对超额收益做 t 检验。
8. MVP 阶段避免太多可调参数。

## 8. 统计显著性

会检验：

- 组合平均日收益是否显著大于 0。
- 组合相对 XLK 的超额收益是否显著大于 0。
- 组合相对 QQQ 的超额收益是否显著大于 0。

注意：

- 金融收益率噪声很大。
- 收益率分布通常不是严格正态分布。
- 不显著不代表策略一定没用。
- 显著也不代表未来一定有效。
- Bootstrap 区间比单纯 t 检验更稳健。

## 10. DeepSeek、LangChain 和 Dify 的选择

推荐路线：

```text
Python 分析引擎 -> Streamlit Dashboard -> DeepSeek 报告生成 -> 可选 LangChain -> 可选 Dify
```

原因：

- 核心计算应该由 Python 完成。
- DeepSeek 负责解释已经算好的结果。
- LangChain 适合后续做工具调用、SQL 查询助手、多步骤报告 agent。
- Dify 适合最后做展示层或工作流包装，不适合作为量化计算核心。

当前 `config/.env` 中 DeepSeek 配置：

```text
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-pro
DEEPSEEK_THINKING=disabled
```

## 13. 表述边界

这个项目是投资研究辅助系统，不应该声称能保证收益。

建议使用：

- “回测结果显示……”
- “样本外结果表明……”
- “该结果与……一致”
- “证据不足以证明……”
- “该策略存在……风险”

避免使用：

- “该策略一定跑赢”
- “该股票一定上涨”
- “模型证明这是最优组合”
- “大模型预测未来走势”
